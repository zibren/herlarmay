# Herlarmay 使用手册

## 概述

Herlarmay 是一款专为 Android 设计的强提醒应用。与普通提醒软件不同，Herlarmay 的核心特点是**即使在应用后台进程被系统杀死后，提醒依然会准时触发**，工作方式类似系统闹钟。

所有提醒（任务）数据均自动保存在设备本地，退出应用后数据不丢失，下次打开时所有已生效的任务自动恢复。

---

## 核心功能

### 1. 自定义提醒时间

- **精确到分**：可设置任意日期时间的提醒，支持时、分精度。
- **一次性提醒**：指定未来某个时间点触发一次。
- **周期性提醒**：按天、周、月重复，例如"每周一早上 9:00"。
- **一次性提醒到期间自动转为周期性**：如需要可设置提醒到期后自动续期。

### 2. 高自由度提醒间隔

- **固定间隔**：每隔 N 分钟/小时/天提醒一次（例如：每隔 30 分钟、每隔 2 小时）。
- **不等间隔**：自定义多个间隔时间组成序列，循环执行。例如：
  - 第 1 次提醒 → 等待 30 分钟
  - 第 2 次提醒 → 等待 1 小时
  - 第 3 次提醒 → 等待 2 小时
  - 然后循环此序列
- **智能间隔**：基于一天中的时段自动调整间隔（如工作时间每 30 分钟，休息时间每 2 小时）。
- **随机间隔**：在设定的最小和最大间隔之间随机触发，适合需要不可预测性提醒的场景（如习惯养成、站立提醒）。

### 3. 强提醒（闹钟级）

Herlarmay 通过以下机制实现**进程被杀死依然能提醒**：

| 机制 | 说明 |
|------|------|
| **`AlarmManager` + `SCHEDULE_EXACT_ALARM`** | 使用 Android 系统级闹钟服务，独立于应用进程运行 |
| **`Foreground Service`** | 在需要时以前台服务运行，提高进程存活率 |
| **`BOOT_COMPLETED` 广播** | 设备重启后自动恢复所有未触发的提醒 |
| **闹钟式提醒** | 提醒时全屏显示 + 震动 + 自定义铃声 + 持续响铃直到手动关闭 |

> **注意**：Android 12+ 上需要用户授予"精确闹钟"权限（`SCHEDULE_EXACT_ALARM`），首次创建提醒时应用会引导您开启。

## 快速开始

### 创建第一个提醒

1. 打开 Herlarmay，点击主界面右下角的 **＋** 按钮。
2. **设置提醒时间**：选择日期和时间。
3. **选择提醒类型**：
   - **一次性**：仅在该时间触发一次。
   - **重复**：按天/周/月重复。
   - **间隔**：设置固定或可变间隔。
4. **设置提醒内容**：输入提醒标题和详细描述。
5. **配置提醒强度**：
   - 选择铃声
   - 开启/关闭震动
   - 是否全屏显示
6. 点击 **保存**，提醒即创建完成。所有提醒数据会自动保存到本地数据库，退出应用后不会丢失，下次打开时自动恢复。

### 管理提醒列表

在主界面可以看到所有已创建的提醒，每个提醒卡片显示：

- 提醒时间/间隔
- 提醒标题
- 下次触发时间
- 开关状态（可随时启用/禁用）

左滑删除，点击进入编辑。所有操作（启用/禁用、修改、删除）均自动保存到本地数据库，无需手动保存。

## 提醒模式详解

### 间隔提醒设置

1. 新建提醒时，选择 **间隔提醒** 模式。
2. **固定间隔**：在"间隔时间"输入框中填入数值，选择单位（分钟/小时）。
3. **不等间隔**：
   - 点击"添加间隔"逐一添加多个时间段。
   - 设置各间隔的先后顺序。
   - 可拖拽排序。
   - 设置是否循环。
4. **开始时间**：设置首次提醒的开始时间（立即开始或指定时间）。

### 示例场景

| 场景 | 设置方式 |
|------|----------|
| 喝水提醒（每隔 1 小时） | 固定间隔 → 1 小时 → 每日 8:00-22:00 |
| 番茄工作法（25 分钟工作 + 5 分钟休息） | 不等间隔 → [25 分钟, 5 分钟] → 循环 |
| 考试复习（间隔逐渐拉长） | 不等间隔 → [30 分钟, 1 小时, 2 小时, 4 小时] → 非循环 |
| 久坐提醒（随机打断） | 随机间隔 → 最小 20 分钟 → 最大 40 分钟 |

## 强提醒工作机制

### 提醒触发时的界面

当提醒触发时，无论手机处于何种状态：

1. **屏幕点亮**（如果处于锁屏状态）
2. **全屏显示提醒界面**，覆盖锁屏
3. **播放铃声**（可自定义，使用系统闹钟音量通道）
4. **震动**（可自定义震动模式）
5. **持续提醒**，直到您手动点击"确认"或"稍后提醒"

### 关闭提醒

- **确认**：点击"确认"关闭当前提醒。
- **稍后提醒**：延迟 5/10/30 分钟后再次提醒。
- **关闭所有**：批量关闭所有同时触发的提醒。

## 数据存储与持久化

Herlarmay 将所有任务（提醒）数据保存在设备本地，确保退出应用或重启设备后数据不丢失。

### 数据模型

在 Herlarmay 中，**任务即提醒**。每个任务包含以下信息：

| 字段 | 说明 |
|------|------|
| **标题与描述** | 提醒时显示的内容 |
| **触发时间** | 首次提醒的日期与时间 |
| **重复规则** | 一次性、按天/周/月重复、固定/不等/随机间隔 |
| **提醒强度** | 铃声、震动、全屏显示等配置 |
| **开关状态** | 启用或禁用 |
| **下次触发时间** | 动态计算的下一次提醒时间 |

### 本地存储方式

| 存储内容 | 技术方案 | 说明 |
|---------|---------|------|
| 提醒数据 | Room 本地数据库 | 所有任务信息持久化到 SQLite |
| 应用设置 | DataStore | 用户偏好、主题、默认铃声等 |
| 已触发日志 | Room 本地数据库 | 提醒历史记录 |

### 数据生命周期

1. **创建/编辑时** → 自动写入 Room 数据库，同时通过 AlarmManager 注册系统闹钟
2. **应用退出时** → 数据已持久化，无需额外操作
3. **应用启动时** → 自动从 Room 数据库加载所有已生效的提醒，恢复界面状态
4. **设备重启时** → `BOOT_COMPLETED` 广播触发，应用从数据库读取所有未触发的提醒，重新注册 AlarmManager 闹钟
5. **卸载时** → 数据库和所有本地文件一并删除

> **提醒（任务）数据仅保存在应用私有目录，其他应用无法访问，无需网络连接。**

## 权限说明

Herlarmay 需要以下权限才能正常工作：

| 权限 | 用途 |
|------|------|
| `POST_NOTIFICATIONS` (Android 13+) | 发送通知栏提醒 |
| `SCHEDULE_EXACT_ALARM` (Android 12+) | 精确闹钟权限，确保准时提醒 |
| `USE_EXACT_ALARM` | 精确计时权限 |
| `FOREGROUND_SERVICE` | 前台服务，提高提醒可靠性 |
| `RECEIVE_BOOT_COMPLETED` | 重启后恢复提醒 |
| `VIBRATE` | 提醒时震动 |
| `WAKE_LOCK` | 点亮屏幕 |
| `READ_EXTERNAL_STORAGE` (部分版本) | 自定义提醒铃声 |

## 常见问题

### Q: 提醒不响怎么办？

1. 确认已授予"精确闹钟"权限（`SCHEDULE_EXACT_ALARM`）。
2. 检查手机是否开启了"省电模式"或"勿扰模式"，部分厂商的系统会限制闹钟。
3. 确认提醒开关处于开启状态（绿色）。
4. 在系统设置中将 Herlarmay 加入"受保护应用"或"自启动白名单"（不同厂商设置位置不同，一般位于：设置 → 应用 → Herlarmay → 省电策略）。

### Q: 不同手机厂商的兼容性如何？

- **小米 MIUI**：需要在"授权管理 → 自启动管理"中开启 Herlarmay。
- **华为 EMUI**：需要在"应用启动管理"中关闭"自动管理"，并手动允许自启动。
- **OPPO ColorOS**：需要在"电池 → 应用耗电管理"中允许唤醒。
- **vivo Funtouch OS**：需要在"电池 → 后台耗电管理"中允许高耗电。

如遇到问题，请参考各厂商的具体设置说明。

### Q: 不等间隔可以设置多少个？

理论上不限数量，但建议不超过 50 个以免列表过长难以管理。

### Q: 任务（提醒）数据会丢失吗？

正常情况下不会。所有提醒数据保存在应用内部的 Room 数据库中，退出应用、系统重启都不会导致数据丢失。唯一会丢失数据的情况是卸载应用或手动清除应用数据。

### Q: 应用被卸载后提醒还会响吗？

不会。提醒数据保存在应用内部，卸载应用后所有数据库和本地文件一并清除，已注册的系统闹钟也会失效。
